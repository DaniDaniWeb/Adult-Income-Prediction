# Adult Income Prediction

Небольшой учебный проект по бинарной классификации на UCI Adult Income Dataset. Цель — предсказать, превышает ли годовой доход человека $50K, и заодно закрепить EDA, preprocessing, `Pipeline`, cross-validation и проверку на data leakage.

Это учебная работа, а не инструмент для реальных решений о найме, зарплате, кредитовании или доступе к услугам.

## Данные

Источник: [UCI Adult Income Dataset](https://archive.ics.uci.edu/dataset/2/adult).

В проекте использован файл `data/raw/adult.data`. В нём есть данные переписи населения США: возраст, образование, занятость, семейное положение, рабочие часы и другие признаки.

В папке `data/raw/` также лежат:

- `adult.test` — исходный файл UCI; он не участвовал в выборе модели;
- `adult.names` — описание полей датасета.

Target `income` был приведён к виду:

- `0` — `<=50K`;
- `1` — `>50K`.

Положительный класс занимает примерно 24% набора, поэтому одной accuracy здесь недостаточно. Основная метрика в проекте — F1 для класса `>50K`.

## Что было сделано

- В `?` распознаны пропуски и заменены на `pd.NA`.
- Для числовых признаков используется заполнение медианой, для категориальных — наиболее частым значением и one-hot encoding.
- Все преобразования находятся внутри `Pipeline`, поэтому imputer и encoder не видят test set во время обучения.
- `fnlwgt` исключён: это вес наблюдения Census, а не обычный признак человека.
- Из пары `education` и `education_num` оставлен `education_num`, чтобы не дублировать одну и ту же информацию.
- Полные дубликаты не удалялись: без идентификатора нельзя уверенно считать, что это ошибочные записи.

## Как оценивались модели

Сначала рабочий набор был один раз разделён на train и test:

```text
test_size=0.2
stratify=y
random_state=42
```

На training data модели сравнивались с помощью 5-fold `StratifiedKFold`. Test set не использовался ни при выборе модели, ни при настройке preprocessing.

Сравнивались три модели:

- `DummyClassifier(strategy="most_frequent")` как baseline;
- `RandomForestClassifier`;
- `XGBClassifier`.

## Результаты

### Cross-validation на training data

| Model | Mean accuracy | Mean precision | Mean recall | Mean F1 |
| --- | ---: | ---: | ---: | ---: |
| Dummy | 0.759 | 0.000 | 0.000 | 0.000 |
| Random Forest | 0.848 | 0.712 | 0.619 | 0.662 |
| XGBoost | 0.867 | 0.755 | 0.662 | 0.705 |

Precision показывает, как часто прогноз `>50K` оказывается верным, а recall — какую долю людей с реальным доходом `>50K` находит модель.

XGBoost показал лучший F1, поэтому именно его я обучил на всей training-части и один раз проверил на отложенном test set.

| Test accuracy | Precision for `>50K` | Recall for `>50K` | F1 for `>50K` |
| ---: | ---: | ---: | ---: |
| 0.870 | 0.757 | 0.679 | 0.716 |

F1 на test set (0.716) близок к среднему F1 по cross-validation (0.705). Для этого разбиения результат выглядит достаточно стабильным.

Отдельно полезно посмотреть на baseline: accuracy у него 0.759, но F1 равен нулю. Он всегда предсказывает самый частый класс и вообще не находит доход `>50K`. Это хороший пример того, почему в несбалансированной задаче нельзя смотреть только на accuracy.

## Ограничения

- Данные старые и не описывают современный рынок труда.
- Модель показывает статистические связи, а не причины высокого дохода.
- Признаки могут быть связаны с чувствительными социальными характеристиками.
- Хорошие метрики не означают, что модель справедлива для разных групп.
- Эту модель нельзя использовать в реальных кадровых, зарплатных или других высокорисковых сценариях.
